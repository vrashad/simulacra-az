# Выбор LLM для симулятора социальной сети (1000 агентов-жителей Баку)

**Дата:** июль 2026 · **Автор тестирования:** Rashad Valiyev

---

## 1. Задача

Симулятор «впрыскивает» пост (рекламу, новость) в синтетическую аудиторию из 1000 демографически реалистичных агентов-азербайджанцев (`profiles_v2/baku_agents_1000.json`, сгенерированы на основе реальной статистики: возраст, доход по квинтилям Q1–Q5, язык, интернет-пенетрация) и отслеживает реакцию на рекламные кампании.

Требования к модели-движку:

1. **Реагировать из профиля агента, а не из собственных приоров** (persona fidelity);
2. **Правдоподобно воспроизводить распределение мнений демографической группы** (group simulation fidelity);
3. Выдерживать масштаб: тысячи вызовов на одну кампанию → критичны цена, скорость, надёжность формата ответа.

## 2. Методология

Использованы два публичных бенчмарка, покрывающих требования 1 и 2, с прогоном через OpenRouter (единый ключ, фиксированный seed 42, temperature 0.0):

| Бенчмарк | Что измеряет | Формат |
|---|---|---|
| **TwinVoice / TwinBench** (ACL'26) | Умение опознать «голос» конкретной персоны по её истории | Дискриминативный: выбор 1 из 4 реплик-кандидатов (случайность = 25%) |
| **SimBench** (Grouped split) | Точность предсказания распределения ответов демографической группы | Verbalized: модель одним вызовом выдаёт JSON-распределение, сравнение с реальными данными опросов (ESS, OpinionQA, Afrobarometer, ISSP, LatinoBarometro) по Total Variation distance |

Кандидаты: `google/gemini-3.5-flash`, `anthropic/claude-sonnet-5`, `openai/gpt-5.5`, `x-ai/grok-4.3` (+ справочно данные автора TwinBench по claude-sonnet-4-6, gpt-5.2, deepseek-v4).

Протокол включал контроль типичных источников ошибок: проверку parse failures в каждом прогоне, фиксацию одинаковых вопросов для всех моделей (общий сэмпл, а не случайные подвыборки), эксперименты с включённым/выключенным reasoning, наращивание выборки для статистической значимости (50 → 250 примеров на TwinBench, 50 → 1500 задач на SimBench).

## 3. Результаты

### 3.1 TwinBench — верность голосу персоны

D1 Social (китайские соцсети Weibo) — решающий прогон на **n=250**, одинаковые вопросы:

| Модель | D1 Accuracy | Режим |
|---|---|---|
| gemini-3.5-flash | **57.6%** | reasoning low |
| gpt-5.5 | 54.4% | reasoning low |
| claude-sonnet-5 | 46–50% (n=50) | оба режима |

Разница gemini vs gpt-5.5 (3.2 п.п. при n=250) статистически незначима — **паритет**. Sonnet-5 стабильно ниже, но на грани значимости.

Важный урок малых выборок: на первых 50 примерах gpt-5.5 «лидировал» с 64%, что на 250 примерах сдулось до 54.4%.

### 3.2 SimBench — верность распределений групп

Основной прогон gemini-3.5-flash на **1500 задачах** (стратифицированная выборка Grouped):

- **SimBench Score: 54.43** (нормировка на uniform-бейзлайн; модель сокращает расстояние до реальных человеческих распределений примерно вдвое против случайного угадывания);
- **0 невалидных ответов из 1500** — идеальная надёжность JSON-формата;
- Total Variation по датасетам: OpinionQA **0.077** · ESS **0.142** · ISSP **0.167** · LatinoBarometro **0.181** · Afrobarometer **0.208**.

Сравнение четырёх моделей на идентичных 50 вопросах:

| Модель | mean TV | invalid | скорость |
|---|---|---|---|
| gpt-5.5 | 0.148 | 0 | — |
| gemini-3.5-flash | 0.158 | 0 | 2.6 с/вызов |
| claude-sonnet-5 | 0.167 | 0 | — |
| grok-4.3 | 0.176 | 0 | 5.2 с/вызов |

Разброс 0.028 при пороге шума ~0.05 (n=50) — **статистическая ничья всех четырёх**.

### 3.3 Побочные выводы тестирования

1. **Reasoning не улучшает симуляцию персон.** Проверено на трёх моделях (gemini: 56% с thinking vs 50% без; sonnet-5: 46% vs 50%; gpt-5.5: 60% high effort vs 64% low) — везде дельта в пределах шума при кратно большей цене. Согласуется с выводом SimBench-статьи: simulation fidelity не растёт с inference-time compute. **Следствие для прода: генерация реакций агентов — без reasoning / минимальный effort.**
2. **Точность падает по мере удаления группы от западных данных:** США (OpinionQA) 0.077 → Европа (ESS) 0.142 → Латинская Америка 0.181 → Африка 0.208. Азербайджан в обучающих данных представлен слабо → ожидаемая ошибка на бакинских группах ~0.2–0.3 TV.
3. **Языковой состав бенчмарков:** TwinBench D1 — китайский, D2/D3 — английский; SimBench — английский. Ни один публичный бенчмарк не покрывает азербайджанский/русский — перенос выводов требует локальной валидации.

## 4. Решение: google/gemini-3.5-flash

Поскольку по качеству фронтир-модели образуют **плато** (статистическая неразличимость на обоих бенчмарках), выбор определили эксплуатационные характеристики:

| Критерий | gemini-3.5-flash | Альтернативы |
|---|---|---|
| Persona fidelity (TwinBench D1, n=250) | 57.6% — лучший численно | gpt-5.5: 54.4%, паритет |
| Group fidelity (SimBench TV) | 0.158–0.170 | 0.148–0.176, паритет |
| Надёжность формата | **0 сбоев на ~1600 вызовов** | 0 у всех (на меньших объёмах) |
| Скорость | **2.6 с/вызов** | grok: 5.2 с |
| Цена | **~$0.0037/задача**; полный SimBench-прогон 1500 задач ≈ $5.5 | gpt-5.5 / sonnet-5 / grok — кратно дороже за счёт обязательного reasoning |

**Итог: при равном качестве gemini-3.5-flash даёт лучшую экономику масштабирования на 1000 агентов × множество кампаний.** Резервный вариант — Qwen на собственном инференсе (Salad/Vast), если симуляция станет постоянным инструментом и экономика self-hosting перевесит API.

## 5. Ограничения и обязательные следующие шаги

Публичные бенчмарки ответили на вопрос «какая модель» — но не «насколько симуляция совпадёт с реальными бакинцами». Симуляция на текущем этапе — **компас, а не линейка**: направлениям («кампания А лучше Б», «молодёжь отреагирует позитивнее») доверять можно, точным процентам — нет (ожидаемая ошибка на азербайджанских группах ~20–30% вероятностной массы).

До принятия бизнес-решений по симуляции:

1. **Калибровка на реальных данных:** прогнать через симуляцию 3–5 прошлых кампаний Birmarket с известными откликами по сегментам; критерий доверия — воспроизведение ранжирования (какая кампания/сегмент откликнулись сильнее).
2. **Собственный AZ-eval:** adherence реакций профилю (LLM-судья, не участвующий в тесте), доля stereotype fallback, «нулевой» контроль (реакция без профиля не должна совпадать с реакцией с профилем), инконгруэнтные агенты как стресс-тест.
3. **Контроль хорового однообразия:** distinct-метрика (эмбеддинги + кластеризация) на каждый впрыснутый пост — 1000 агентов не должны схлопываться в 5–7 архетипных реакций; температура генерации 0.8–1.0.
4. Известные риски сетапа: LLM-сгенерированные персоны вносят собственные смещения; socio-demographic prompting ненадёжен для субъективных суждений (подтверждено литературой) — оба риска закрываются пунктами 1–2.
